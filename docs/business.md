# 业务文档：聚合热闻

## 1. 产品定位

聚合热闻是一款“新闻文本中台 + AI 评论员 + 语音播报器”的移动端应用。它不是替代媒体原创报道，而是帮助用户更快地发现新闻、理解新闻，并在无法看屏幕时继续收听。

## 2. 核心业务流程

1. 用户打开应用。
2. 系统加载已启用抓取范围并刷新新闻。
3. 系统将新闻稿按时间汇总展示。
4. 用户按平台、关键词、收藏、未读或范围标签筛选新闻。
5. 用户选择一条新闻，进入详情页查看文本稿。
6. 用户可复制、分享、打开原文、收藏或标记已读/未读。
7. 用户点击“锐评”，应用调用用户配置的大模型生成短评。
8. 用户点击“播报”，应用使用本地或远程 TTS 播放新闻稿；点击“连续播报”可收听当前列表队列。
9. 用户进入“设置”管理新闻源、锐评模型、语音播报和应用更新。

## 3. 抓取范围策略

默认范围聚焦摸摸鱼聚合热榜：

- 摸摸鱼聚合热榜：`https://www.momoyu.cc/api/hot/list?type=0`

该接口会返回多个平台的热榜数据，首页直接按返回的平台名称分类展示。

自定义范围字段：

- 名称：用户可读的媒体或栏目名称。
- 范围标签：综合、国内、国际、财经、科技等。
- 类型：`momoyu_hot`、`rss`、`cctv_jsonp`、`baidu_hot`、`toutiao_hot`、`kr36_flash`、`thepaper_hot`、`v2ex_hot`、`sspai_home`。
- URL：抓取入口。
- 启用状态：是否参与刷新。

保存规则：

- URL 必须以 `http://` 或 `https://` 开头。
- 类型只能是应用支持的解析器类型：`momoyu_hot`、`rss`、`cctv_jsonp`、`baidu_hot`、`toutiao_hot`、`kr36_flash`、`thepaper_hot`、`v2ex_hot`、`sspai_home`。
- 同一个 URL 不允许重复保存。
- URL 为空时弹窗保持打开并提示用户补全。
- 用户可对单个来源执行“测试”，检查当前配置是否能抓到新闻。
- 用户可一键恢复默认来源，清理错误自定义配置。
- 用户可导出当前来源配置为 JSON 到剪贴板，用于备份、迁移或发给其他设备。
- 用户可粘贴 JSON 导入来源配置；导入时可选择合并到现有范围或替换全部范围。
- 合并导入时，同 URL 或同 ID 的旧配置会被导入配置覆盖，避免重复来源。
- 每次刷新和单源测试都会更新来源诊断，范围页展示最近一次成功/失败、抓取条数、检测时间和失败原因。
- 当部分来源失败时，成功来源的新闻仍会展示；失败来源可在范围页快速定位。

## 4. 大模型业务规则

新闻锐评的默认要求：

- 100 字以内。
- 冷静、犀利但负责任。
- 指出公共价值、潜在影响和需要继续观察的点。
- 避免未经证实的断言。

用户可替换：

- Endpoint。
- API Key。
- Model。
- Prompt。
- 用户可在设置页点击“测试锐评接口”，系统会使用当前表单配置发送短测试请求，并在状态栏显示成功摘要或错误原因。

该设计可以接入 OpenAI、Azure OpenAI、通义千问兼容网关、智谱兼容网关、DeepSeek 兼容网关、小米 MiMo Token Plan 等 OpenAI-compatible 服务。

小米 MiMo Token Plan 接入规则：

- Base URL 以控制台为准；新加坡集群示例为 `https://token-plan-sgp.xiaomimimo.com`，应用会自动补齐 `/v1/chat/completions`。
- API Key 格式通常为 `tp-xxxxx`，应用使用 `api-key` 请求头。
- 推荐锐评模型可配置为 `mimo-v2.5-pro`。

## 5. 语音播报业务规则

本地 TTS：

- 作为远程语音不可用时的兜底模式。
- 不需要 API Key。
- 依赖 Android 系统语音引擎和中文语音包。
- 支持对当前筛选后的新闻列表连续播报，最多取前 20 条，并在播报时标记为已读。
- 设置页自检会播放一句本地测试语音，用于确认系统语音包可用。

远程 TTS：

- 默认使用小米 MiMo Chat Audio。
- 适合更自然的播报音色。
- 用户配置 Endpoint、API Key、Model、Voice。
- 当前实现同时支持 OpenAI-compatible Speech API 的二进制音频返回，以及 Xiaomi MiMo TTS 的 Chat Completions `audio.data` base64 音频返回。
- 小米 MiMo TTS 可配置 `mimo-v2.5-tts` 与内置音色，如 `mimo_default`、`冰糖`、`茉莉`、`苏打`、`白桦`、`Mia`、`Chloe`、`Milo`、`Dean`。
- 当前远程 TTS 用于单条新闻播报，连续队列使用本地 TTS 以避免批量生成音频造成费用和延迟不可控。
- 设置页自检会使用当前远程语音配置生成一段短测试音频并播放；远程不可用时自动回退系统 TTS。

## 6. 商业化与合规建议

- 内容授权：与媒体源签订 API 或 RSS 使用协议。
- 缓存策略：仅缓存必要字段，尊重源站 robots、版权声明和缓存时间。
- 频率控制：端侧抓取适合原型；正式版建议服务端统一抓取、清洗、去重和限频。
- 隐私保护：API Key 当前存储在本地 SharedPreferences；正式版建议加密存储或使用服务端代理。
- 内容安全：AI 锐评需加入事实性约束、敏感内容策略和人工可审计日志。

## 7. 当前增强能力

- 最近 240 条新闻会缓存在本地，弱网或源站失败时仍可继续阅读，并尽量保留不同范围可筛选的数据。
- 关键词筛选覆盖标题、摘要、来源、范围，适合快速查找某一主题。
- 平台分类来自摸摸鱼返回的各平台名称，可在首页横向切换。
- 范围标签筛选可按来源配置中的范围快速切换新闻池，并与关键词、收藏/未读视图叠加。
- 新闻页会统计当前可见结果数量、未读数量和收藏数量，便于用户判断当前筛选池。
- 用户可将当前可见结果批量标记为已读或未读，批量操作只影响当前关键词、范围和视图条件下的稿件。
- 用户可复制当前可见标题清单，便于转发给团队、整理选题或快速沉淀工作记录。
- 新闻支持收藏、取消收藏、标记已读/未读，并提供全部/收藏/未读视图。
- “连续播报”可将当前可见新闻作为收听队列，适合通勤或免手持场景。
- 当前新闻稿可以复制到剪贴板或走系统分享面板。
- 有原文链接的稿件可直接跳转浏览器查看来源页面。
- 来源管理支持单源测试和恢复默认范围，便于排查自定义源问题。
- 来源管理支持导入/导出 JSON，方便批量维护媒体源配置。
- 来源诊断能帮助用户发现失效 RSS、源站限流、接口结构变化等问题。
- 模型接口自检能帮助用户在正式锐评或播报前发现 Key、Endpoint、Model、Voice 配置问题。
