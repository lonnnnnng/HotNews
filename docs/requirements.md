# 需求文档：聚合热闻

## 1. 背景

用户希望在手机端实时查看来自国内知名媒体的热门新闻汇总，并在阅读时获得大模型锐评和语音播报能力。应用需要允许用户调整抓取范围，也允许自行配置新闻锐评模型和语音模型。

## 2. 目标用户

- 高频新闻阅读用户：希望快速浏览多个媒体的重点消息。
- 内容运营/研究人员：需要快速汇总新闻稿并获得评论视角。
- 通勤/驾车用户：希望通过语音收听新闻稿。

## 3. MVP 功能

| 编号 | 功能 | 说明 |
| --- | --- | --- |
| R1 | 新闻汇总 | 从已启用新闻源抓取新闻，按时间汇总展示标题、摘要、来源、范围和时间。 |
| R2 | 默认抓取范围 | 默认启用摸摸鱼聚合热榜，并从其返回数据中按平台分类展示国内热门站点内容。 |
| R3 | 自定义范围 CRUD | 用户可新增、编辑、启用/停用、删除抓取范围。 |
| R4 | 新闻稿展示 | 选择新闻后生成“标题 + 摘要 + 来源 + 时间”的新闻稿文本。 |
| R5 | 大模型锐评 | 用户配置 OpenAI-compatible Chat Completions 接口后，对当前新闻稿生成短评，支持普通 Bearer 鉴权和 Xiaomi MiMo Token Plan `api-key` 鉴权。 |
| R6 | 语音播报 | 支持 Android 本地 TTS 播报当前新闻稿和锐评。 |
| R7 | 远程 TTS | 用户可配置 OpenAI-compatible Speech 接口生成并播放语音，也可配置 Xiaomi MiMo TTS，通过 Chat Completions 返回的 `audio.data` 生成播报音频。 |
| R8 | 实时刷新 | 打开应用自动刷新，前台每 10 分钟自动刷新，支持手动刷新。 |
| R9 | 关键词筛选 | 支持按标题、摘要、来源、范围筛选当前新闻列表，并记住最近一次筛选词。 |
| R10 | 本地缓存 | 刷新成功后缓存最近 240 条新闻，网络失败时可展示本地缓存。 |
| R11 | 复制/分享/原文 | 当前新闻稿支持复制到剪贴板、系统分享和打开原文链接。 |
| R12 | 来源校验 | 自定义来源保存时校验 URL、类型和重复 URL，避免无效配置污染抓取结果。 |
| R13 | 收藏/未读 | 新闻支持收藏、取消收藏、标记已读/未读，并提供全部/收藏/未读视图。 |
| R14 | 来源诊断 | 单个来源支持连通性测试，来源列表支持一键恢复默认范围。 |
| R15 | 连续播报 | 支持对当前可见新闻队列进行连续本地 TTS 播报，并自动标记已读。 |
| R16 | 平台分类 | 首页按新闻平台分类展示，支持全部平台和单平台切换，并保持分类横向滚动位置。 |
| R17 | 范围导入导出 | 支持将抓取范围导出为 JSON 到剪贴板，也支持粘贴 JSON 导入；导入可合并或替换，并执行字段校验。 |
| R18 | 来源刷新诊断 | 每次刷新或单源测试后记录来源成功/失败、抓取条数、检测时间和错误原因，并在范围页展示。 |
| R19 | 范围标签筛选 | 支持按新闻范围标签筛选新闻池，并联动新闻列表和连续播报。 |
| R20 | 模型接口自检 | 设置页支持测试新闻锐评接口和语音接口，本地 TTS 可直接播放测试语音，远程语音可生成短测试音频。 |
| R21 | 可见结果批量操作 | 新闻页支持统计当前可见稿件，并对当前可见结果批量标已读/未读、复制标题清单。 |

## 4. 非功能需求

- 可用性：无 API Key 时仍可浏览新闻、使用本地 TTS，并在网络失败时查看最近缓存。
- 可配置：新闻源、锐评模型、TTS 模型均可在端侧设置。
- 可诊断：新闻源、锐评接口和语音接口均提供端侧自检反馈。
- 可维护：默认源与自定义源使用统一数据模型。
- 合规：默认公开源仅用于原型验证；正式上线前需处理版权、授权、频率限制和隐私政策。
- 凭据安全：文档和代码不保存真实 API Key；调试时仅写入设备本地应用数据。

## 5. 后续增强

- 新闻去重聚类：引入更强的语义向量或服务端聚类，将多个媒体对同一事件的报道合并为事件卡片。
- 多维筛选：按地区、行业、媒体、关键词、收藏/未读、负面/突发标签筛选。
- 筛选组合记忆：记录用户常用的关键词、范围和视图组合，一键切换。
- 来源模板市场：提供财经、科技、国际、地方等预设模板包，一键导入。
- 热榜适配扩展：对需要登录或验证码的网站使用服务端授权抓取或第三方合规数据服务，不在端侧硬抓易失效接口。
- 来源健康看板：统计连续失败次数、平均延迟和最近 24 小时可用性。
- 推送摘要：支持早晚报、突发新闻推送和静默时段。
- 批量收藏：支持对当前可见稿件批量收藏、取消收藏，并提供二次确认。
- 离线缓存：保留最近 7 天稿件与语音缓存。
- 可信度提示：标注来源、转载链路、未经证实信息和更新状态。
